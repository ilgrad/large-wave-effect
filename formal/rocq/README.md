# Machine-checked certificate (Rocq)

`Mod4Criterion.v` is generated by `../../exact/gap/export_rocq.g` and checked by the
[Rocq Prover](https://rocq-prover.org) (formerly Coq), version 9.x. It is self-contained: it uses
only the standard library (`ZArith`, `List`), no MathComp, no `opam`.

```bash
# Fedora: dnf install rocq-prover
gap -q --nointeract exact/gap/export_rocq.g   # regenerate Mod4Criterion.v
rocq compile formal/rocq/Mod4Criterion.v      # machine-check it
```

## What it certifies

The file defines the mod-4 ceiling criterion directly on integer relation vectors:

```
saturates N B := forallb node0 B  ||  (Z.even N && forallb half B)
```

(`node0 k := (sum k) mod 4 = 0`, `half k := (sum k + 2 * sum_r r*k_r) mod 4 = 0`), embeds the exact
relation-lattice bases `Lambda_N` computed by GAP for every `N` in the range, and proves by
reflection (`vm_compute`)

```
Theorem mod4_criterion_matches_prime_2pow :
  map (fun p => saturates (fst p) (snd p)) allcerts = expected.
```

i.e. the criterion's verdict on the GAP lattices equals the prime/`2^m` classification, with **no
floating point** anywhere in the proof.

## What it does *not* certify (honest scope)

- It does **not** prove that the embedded vectors are genuine frequency relations — that the rows of
  `Lambda_N` actually satisfy `sum_r k_r * 2 sin(pi r/N) = 0`. That rests on the cyclotomic theory
  (proved on paper, checked numerically by GAP/PARI), which is *not* formalized here.
- It does **not** prove the general classification `A_N = U_N <=> N` prime or `2^m`; it certifies the
  criterion evaluation on a finite, explicit set of `N` only.

In short: Rocq removes any doubt about the *combinatorial evaluation* of the criterion on the
certificates; the cyclotomic input is supplied, exactly, by GAP and PARI.
